# 에러 UI 테스트 가이드

## 현재 상태

에러를 강제로 발생시키도록 코드가 수정되었습니다.

### 수정된 파일

1. **`src/entity/post/api/posts.ts`** (5번 줄)
   - 게시물 목록 로딩 시 에러 발생
   - 페이지를 새로고침하면 게시물 목록 영역에 에러 메시지 표시

2. **`src/entity/comment/api/comments.ts`** (4번 줄)
   - 댓글 목록 로딩 시 에러 발생
   - 게시물 상세 모달을 열면 댓글 영역에 에러 메시지 표시

## 테스트 방법

### 1. 게시물 목록 에러 확인

1. 개발 서버 실행 중인지 확인
   ```bash
   pnpm dev
   ```

2. 브라우저에서 페이지 열기
   - `http://localhost:5173` (또는 개발 서버 주소)

3. 페이지 새로고침 (F5)
   - 게시물 목록 영역에 빨간색 에러 박스가 표시됩니다
   - "게시물을 불러오는 중 오류가 발생했습니다" 메시지 확인
   - "다시 시도" 버튼 확인

### 2. 댓글 목록 에러 확인

1. 게시물 목록에서 아무 게시물이나 클릭
2. 게시물 상세 모달이 열림
3. 댓글 영역에 빨간색 에러 박스가 표시됩니다
   - "댓글을 불러오는 중 오류가 발생했습니다" 메시지 확인
   - "다시 시도" 버튼 확인

## 에러 메시지 스타일

에러 메시지는 다음과 같이 표시됩니다:

```
┌─────────────────────────────────────────────┐
│ ⚠️ 오류가 발생했습니다                      │
│                                             │
│ 테스트용 에러: 게시물을 불러올 수 없습니다  │
│                                             │
│ [ 다시 시도 ]                               │
└─────────────────────────────────────────────┘
```

- 빨간색 배경 (`bg-red-50`)
- 빨간색 테두리 (`border-red-200`)
- 경고 아이콘 (AlertCircle)
- 재시도 버튼

## ⚠️ 중요: 테스트 후 복구

**테스트가 끝나면 반드시 에러 코드를 제거하세요!**

### 복구 방법

1. **`src/entity/post/api/posts.ts`** 파일 열기
2. 5-6번 줄의 다음 코드를 **삭제**:
   ```typescript
   // 🔴 테스트용: 에러 UI 확인을 위한 임시 에러 (테스트 후 제거하세요!)
   throw new Error("테스트용 에러: 게시물을 불러올 수 없습니다")
   ```

3. **`src/entity/comment/api/comments.ts`** 파일 열기
4. 4-5번 줄의 다음 코드를 **삭제**:
   ```typescript
   // 🔴 테스트용: 에러 UI 확인을 위한 임시 에러 (테스트 후 제거하세요!)
   throw new Error("테스트용 에러: 댓글을 불러올 수 없습니다")
   ```

5. 파일 저장 후 페이지 새로고침
6. 정상적으로 데이터가 로드되는지 확인

## 대안 방법 (코드 수정 없이)

코드를 수정하지 않고 테스트하고 싶다면:

### 브라우저 개발자 도구 사용

1. **F12** 키로 개발자 도구 열기
2. **Network** 탭 선택
3. 상단의 **Throttling** 드롭다운에서 **"Offline"** 선택
4. 페이지 새로고침
5. 에러 메시지 확인
6. 테스트 후 **"No throttling"**으로 복구

이 방법은 코드 수정 없이 바로 테스트할 수 있습니다.
